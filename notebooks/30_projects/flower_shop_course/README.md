# 用 Python 經營一家花店：課程想法整理

目的：用「開花店」這個真實情境，把 Python 入門、資料結構、模組化與基礎分析串起來，訓練學生把現實問題轉成可計算模型。

---

## 課程定位
- **對象**：高中生，零基礎起步
- **技術主線**：Python 基礎 → 資料結構 → 函式/模組化 → 簡單分析
- **思維主線**：業務場景 → 資料結構 → 決策與優化

---

## 課程總體結構（6 階段）

### 1. Python 基礎與計算思維
- 什麼是程式？為什麼花店需要 Python
- 變數、資料型別（int/float/str）
- 輸入輸出：模擬顧客下單
- 條件判斷：庫存/折扣/預算
- 迴圈：批量統計、一天營業流程
- 小練習：模擬「一天賣花」

### 2. 花店業務建模（把現實變成資料）
- list：一類花、訂單清單
- dict：一朵花（名稱/進價/售價/庫存）
- 組合結構：花材列表 + 花材字典、訂單列表 + 訂單字典
- 抽象訓練：屬性 vs 行為

### 3. 進銷存與訂單系統
- 進貨系統：新花入庫 / 補貨 / 庫存更新
- 銷售系統：購買流程 / 扣庫存 / 累計銷售額
- 訂單系統：編號生成 / 結構設計 / 多訂單管理
- 例外處理：庫存不足、輸入錯誤、避免崩潰

### 4. 函式與模組化（從能跑到像系統）
- 為什麼需要函式：一個函式 = 一個業務動作
- 核心函式設計：
  - add_flower()
  - sell_flower()
  - create_order()
  - show_inventory()
- 模組化思維：庫存模組 / 訂單模組 / 財務模組

### 5. 成本管理與財務分析
- 成本概念：進貨 / 器材 / 固定 vs 可變
- 收入與利潤：銷售額 / 毛利 / 簡化淨利
- Python 統計：總成本 / 總收入 / 利潤率
- 資料分析初步：最賺錢花種 / 銷量高峰 / 庫存周轉率（簡化）

### 5-1. 期貨概念（花店版風險管理）
- 什麼是期貨：**先約定未來的價格與數量**
- 花店情境：節日前先「鎖定進貨價格」，避免花材大漲
- 風險觀念：鎖價降低不確定，但可能錯過更低價

**適合的資料結構**
- `NamedTuple`：固定合約資料（品項、數量、約定單價、交付日）
- `dataclass`：合約狀態（已交付/未交付、已付款/未付款）
- `dict`：用合約編號快速查詢

### 6. 綜合專案實戰（完整花店系統）
- 需求分析：系統邊界與必要功能
- 架構設計：資料結構總覽 + 功能模組圖
- 編碼實作：完整花店系統
- 測試與優化：模擬真實使用、改進體驗

## 資料結構清單（必要 / 常用 / 好用 / 優雅）
以下作為後續課件編排依據，依常見度與價值分類。

### 必要（一定會用）
- **list**：商品清單、訂單列表
  - 例：`flowers = ["玫瑰", "百合"]`
- **dict**：單筆商品/訂單資料
  - 例：`{"name": "玫瑰", "price": 30, "stock": 20}`
- **set**：去重、快查
  - 例：不同花種清單、避免重複進貨
- **tuple**：固定格式輸入
  - 例：`(flower_id, qty)`

### 常用（會大量出現）
- **list of dict**：整體庫存
  - 例：`[{"name": "玫瑰", ...}, {"name": "百合", ...}]`
- **dict of dict**：用名稱當 key 的快速查找
  - 例：`inventory["玫瑰"]["stock"]`
- **dict of list**：依日期分類的訂單
  - 例：`orders_by_day["2026-02-01"] = [order1, order2]`

### 好用（提升可讀性與維護性）
- **NamedTuple**：不可變、欄位清楚
  - 例：`Flower(name, cost, price, stock)`
- **dataclass**：可變、適合狀態更新
  - 例：`Order(id, items, total)`

### 優雅（進階但實用）
- **defaultdict**：簡化累加邏輯
  - 例：`sales_by_flower["玫瑰"] += qty`
- **Counter**：快速統計銷量
  - 例：最熱銷花種
- **deque**：模擬排隊訂單
  - 例：先進先出處理
- **heapq**：取最高利潤/最低庫存
  - 例：優先補貨的花材

## 新增環節：節日訂單處理全流程
目的：把「高峰期」的真實業務壓力轉成可計算流程與規則。

### 流程步驟
1. **預估需求**：根據去年的節日銷量做簡單預估
2. **提前備貨**：鎖定主打花材與包材（花束紙、卡片）
3. **排程與分流**：依時段分單、標記急件
4. **訂單建立**：統一格式（訂單編號、內容、取件/配送）
5. **庫存扣減**：成立訂單即預扣庫存
6. **製作流程**：製作狀態追蹤（待做 / 製作中 / 完成）
7. **出貨與交付**：取件、配送、簽收
8. **退補與缺料處理**：缺花材即改替代品
9. **結算與回顧**：成本、毛利、熱銷款統計

### 建議課件重點
- 以 **list + dict** 建模訂單
- 用 **queue/deque** 模擬排單
- 用 **status 欄位** 追蹤流程
- 以 **Counter** 統計熱銷款

## NamedTuple / dataclass 導入時機
**建議節點**
- **第 2 節後**：學生已熟悉 dict 結構，可開始改用 NamedTuple/dataclass
- **第 3~4 節**：訂單、庫存、狀態更新變多，建議正式切換

**導入方式**
- 先用 NamedTuple 表示「固定資料」：花材、座位、既定訂單
- 再用 dataclass 表示「會變動狀態」：庫存、訂單狀態、營收統計
