## 課程流程
### 18:40 — 19:10 作業檢核與補充講義追蹤
1. 先詢問上週作業完成度，學生表示有寫；實際檢查後發現 comprehension 練習仍卡在課堂錯誤版本，且題目被改錯。
2. 追問 comprehension 補充講義的閱讀情況，確認僅看了開頭十幾分鐘，後半段（包含完整解答與練習引導）完全沒看。
3. 說明：補充講義中已附上 Week08 小練習的完整答案與思路，如果願意先自行準備，就能在課堂上更快進入討論，而不是從空白開始。
4. 正式表達失望與擔心——不是因為「寫不出來」，而是因為「沒有給自己時間去練習」，提醒他這樣的習慣會拖累後面所有進度。

### 19:10 — 19:40 學校課程情況了解與學習方式討論
1. 請學生分享學校週六 Python 課程進度，查看老師使用的 PPT 與課件內容。
2. 學校老師主要以 PPT 投影片搭配現場輸入程式碼授課，部分題目直接截取題庫範例題作為作業。
3. 學生分享同學們如何從題目截圖中「看顏色」偷看正確答案（紅色轉成灰階後顏色較淡），作為小樂趣。
4. 回應：觀察答案顏色固然有趣、代表有觀察力，但真正重要的是能不能靠邏輯與理解寫出解答；也再次提醒「看答案」不等於「會做題目」。
5. 也有觀察到學生對 PPT 某一題（大概是第 17 題）特別感興趣，但因為當下未記下題號，暫時先不改動原有進度安排。


### 19:45 — 20:10 小練習 1：tuple / list 與平均數（回補）
1. 回顧 Week08 的「練習：tuple / list 小試身手」，再次從 `records` 資料結構開始拆解。
2. 帶著學生完成 `first_student`、`others` 的 indexing，並示範如何用 for 迴圈或 comprehension 計算 `averages` 與 `passed` 名單。
3. 大致上思路已經理解，但細節（例如 `round` 的使用、內層 list 的平均計算）仍需參照上方範例才能寫出。
4. 課堂中只完成到「第一個小練習的主要邏輯」，細節留作回家自行補完。


### 20:10 — 20:45 進入 Week09 講義：處理輸入資料
1. 說明本週主題：如何從輸入文字中取出需要的資料，包括整筆輸入、去頭去尾、切成句子或 list 等。
2. 一步一步帶著學生操作，示範 `strip()`、`split()` 等處理方式，並口頭要求他說出每一步得到的資料型態。
3. 將文字輸入轉成 list、再轉成需要的結構；過程中進度明顯偏慢，多數步驟仍需老師提示才敢動手。
4. 小練習 1：實作 `read_couplets_from_lines(lines)`，帶著完成大部分迴圈與型態轉換，但尚未完全獨立完成。


### 20:45 — 21:10 與家長討論進度與期待（師生家長三人）
1. 學生先行離開前，與家長與學生三方約 30 分鐘，整理目前數週來的進度落差與作業完成度。
2. 說明：若只靠每週一次上課、且課後幾乎沒有自主練習，要應付 APCS 題目會非常吃力。
3. 討論之後的上課節奏與時間管理，明確說明不再無限延長課堂時間，避免 21:00 以後還在補進度。
4. 確立未來上課時間需控制在 18:30~20:30，超過的時間教師也需要留給備課與復盤，不再「用加班換進度」。

### 21:10 — 21:50 與家長單獨深談
1. 學生離開後，與家長再單獨討論約 40 分鐘，細部說明最近幾週作業完成度與長期目標落差。
2. 說明目前課程重點仍在打基礎與建立學習習慣，而非只「衝題數」，請家長協助留意孩子回家實際動手時間。
3. 再次強調未來會嚴格控制下課時間，讓課程與教師備課都維持在可長期持續的節奏。

## 現況 / 進度評估
- **作業與預習狀況**：Week07 的「輸入生成菱形」和 Week08 的 comprehension 練習尚未完成，補充講義只看了開頭，尚未建立「先看講義→再提問」的習慣。
- **語法掌握**：在老師帶領下能理解 tuple/list 結構與 `strip`、`split`、`map(int, ...)` 等用法，但獨立實作仍有困難，容易卡在細節。
- **學習態度**：樂於分享學校課堂趣事，但在自己該完成的練習上容易拖延；需要外力提醒才會打開講義或作業。
- **時間管理**：課堂常因補作業與說明進度而被迫延長，實際有效寫程式的時間過短，效率偏低。

## 教學反思
- 過去為了補進度，常主動加長課程時間，反而讓學生感覺「反正老師會陪我寫完」，削弱課後主動練習的必要性。
- 補充講義的篇幅與文字量可能對目前的閱讀習慣來說壓力過大，之後需拆成更小段、搭配勾選式任務，讓他可以一段段完成。
- 面對「自稱有寫作業但實際未完成」的情況，需要更明確、具體的檢核（例如：請拍照、請上傳 Notebook 檔），減少模糊空間。
- 未來要在課前就確認上一週作業是否完成，若未完成則直接在課程一開始處理，而不是壓在課尾才發現進度落差。
