## 課程概況
- 開場先用兩張 Weekly Planner 盤點每週固定行程與可用時段，花了約一小時梳理「可用在 Python 的時間」。

- 探討筆記習慣與英文瓶頸：筆記能力不足、對英文有排斥；透過對話引導他理解同儕「看似不讀書也考得好」背後可能的努力與差異。
- 作業檢核：動態輸入生成圖形程式有寫但邏輯錯誤，暫擱置，計畫 Week11 課堂重寫；補充講義這週有看，提問能回答。
- 針對 set 操作仍不熟悉，臨時加了一段基礎操作與官方文件查詢示範，21:50 左右收課；態度比前一週顯著改善。

## 課程流程
### 18:40 — 19:50 時間規劃與學習習慣談話
1. 提供兩張空白 Weekly Planner，逐格填寫固定課程、社團與可運用的零碎時間。
2. 學生提出每週可再投入 3~4 小時學 Python（不含週三上課），老師引導他訂為 2~3 小時且分散到可維持專注的時段。
3. 討論筆記習慣，發現過去幾乎不寫，決定先用簡短重點式紀錄，而非長篇心得。
4. 轉向英文學習困難：討厭英文、考試常自以為對結果錯、過去補習班聽不懂而放棄；鼓勵以短時間、可理解輸入慢慢重建信心。
5. 對同儕「玩也考得好」的觀感進行拆解，提醒看不到的努力與天賦差異，強調自己需要穩定投入。

### 19:50 — 20:30 作業檢核與延後處理
1. 查看「依使用者輸入動態生成形狀」程式，確認已嘗試撰寫但結果錯誤；先記下需求，預計 Week11 課堂重寫。
2. 重新檢查 comprehension 補充講義：本週有閱讀，詢問重點題目皆能口頭作答，較上週無進度的情況改善。

### 20:30 — 21:10 set 操作補課
1. 發現對 `set` 基礎操作（新增、刪除、交併差）記憶模糊，臨時加段練習與語法講解。
2. 示範如何用 Google / Python 官方文件查指令，鼓勵遇到不熟語法時先查證，再實作。

### 21:10 — 21:50 整理與課後提醒
1. 回顧今日重點：時間規劃、英文心態、set 操作，確認他理解「少量但穩定的練習」目標。
2. 再次提醒下週會在課堂重寫圖形程式，請先想清楚輸入/輸出需求與迴圈結構。

## 教學反思
- 近一小時的規劃談話佔用大量課程時間，之後要預留最少一半時間給實作練習，避免再度延到 21:50。
- 學生缺少筆記與自查框架，應提供極簡模板（重點、問題、下一步）而非長篇紀錄，降低阻力。
- 英文排斥感需要循序漸進引導，可在程式練習中穿插短英文提示或文件閱讀，讓他累積可理解輸入。
- 對作業錯誤的處理要先明確列出「下週要重寫的目標」，避免因延後而淡化重要性。

## 下週計畫
1. 開場 10 分鐘確認 Weekly Planner，敲定每週 2~3 小時的固定練習時段與回報方式。
2. 30~40 分鐘課堂重寫「依輸入動態生成圖形」程式，先手寫邏輯後再上機。
3. 安排 set 基礎操作快測（交併差、`add` / `discard`、型態轉換），檢核是否能獨立完成。
4. 對 comprehension 講義再抽問 1~2 題，確保理解不是短期背誦。
5. 若時間允許，安排 10 分鐘英文/文件閱讀任務（簡短段落），練習從官方文件找語法用法。
6. 嚴格控制在 20:30~20:40 收課，避免持續超時。

## 課後任務（給學生）
- 依 Weekly Planner 填好下週課表與 Python 練習時段，標記預計時長（每次 30~60 分鐘），上課時帶來對照。
- 先在紙上或 Notebook 草擬「動態生成圖形」的輸入/輸出範例與迴圈思路，下週照此重寫。
- 複習 set 基礎操作：`add`/`remove`/`discard`、交併差、`len`、`in`，把不熟的語法寫成 3~5 行範例並記下結果。
- 開始嘗試簡短筆記：今日的 3 個重點、1 個問題、1 個下一步，下次課堂帶來檢視。
- 選一段官方文件（例如 `set` 章節）閱讀 5 分鐘，記錄看不懂的詞，練習查字典或翻譯。
