# 第五週教案安排 W05 Lesson Plan

> 主題：布林語意拆解 × 計數器 × 密碼錯誤次數限制  
> 時間配置：120 分鐘（可依實際狀況微調）

---

## 課前準備
- 收集學生上週的自我填寫練習（倒數追蹤表、條件拆解表、真值表小測驗），預先檢視常見錯誤。
- 印出或上傳「日常語意 ↔ 布林判斷」講義（含填空表格）供課堂填寫。
- 準備 `week05_boolean_counter_lab.ipynb`，內含各段練習與展示程式。
- 確認教室網路、VS Code、Jupyter Notebook 正常運作；備用方案為投影教師端操作。
- 若學生仍不熟悉 `input()`，備好小測或速寫題檢核。

---

## 教學目標
1. 學生能將生活語句轉換為布林變數與真值表，理解 `and` / `or` / `not` 的意義。
2. 學生能在 `for` / `while` 迴圈中使用計數器觀察數值變化，並以表格追蹤狀態。
3. 學生能拆解並實作「密碼錯誤次數限制」邏輯，知道如何用布林判斷與計數器組合。
4. 教師能即時診斷學生對布林條件與迴圈累加的盲點，並提供具體引導。

## 課程流程

### 0. 課前 5 分鐘｜學生自填回饋蒐集
- 收回上週練習單，讓學生標記最不確定的題目。
- 教師快速瀏覽，預判待加強點。

### 1. 19:30–19:45｜暖身檢核（15 分）
1. 快問快答：請學生口頭說明 `range(5, 0, -2)`、`total += i` 的意義。
2. 播放 Notebook 中的「倒數追蹤表」，請學生補寫缺失欄位。
3. 針對錯誤或遲疑的項目立即補充，確保後續可進入主題。
4. 在進入新內容前，逐項檢討並訂正上週學習單：
   - 核對 `range` 倒數追蹤表是否完成，重點說明 `start`/`stop`/`step`。
   - 檢查真值表題目，要求學生解釋 0/1 與布林值的對應。

### 2. 19:45–20:15｜日常語意 → 布林真值表（30 分）
1. 導入 Notebook 第 1 節：說明 A、B 兩個布林變數，各代表一句話中的兩件事情。
2. 逐一示範範例「下雨我帶傘」，帶學生把四種組合寫進真值表並計算 `A and B`、`A or B`、`A xor B`。
3. 每完成一列要求學生說明「這一列代表什麼情境？符合原句嗎？」練習在結果欄寫勾/叉與中文說明。
4. 學生分組完成 B～E 表格，教師巡視並即時提問「哪一列會被判斷為 False？為什麼」。
5. 表格填完後，再整理符合條件的列，寫出對應的 Python 條件語句（例如 `not A or B`）。
6. 執行 Notebook 中的 `truth_table_for` 範例程式，示範如何用程式驗算真值表；請學生替自己的條件函式呼叫一次檢查。

### 3. 20:15–20:35｜計數器與迴圈追蹤（20 分）
1. 使用 Notebook 範例展示以下程式：
   ```python
   snowdays = 0
   for day in ["Mon", "Tue", "Wed", "Thu", "Fri"]:
       if day in {"Wed", "Thu"}:
           snowdays += 1
       print(day, "目前 snowdays =", snowdays)
   ```
2. 請學生同步填寫表格：
   | 回合 | day | 條件成立? | snowdays (前) | snowdays (後) |
3. 介紹常見計數器模式 `count += 1`、`count = count + 1`、`count = count + step`。
4. 延伸至 `while` 迴圈：加入倒數、迴圈終止條件，強調「計數器位置」與「更新時機」。
5. 即時練習：請學生寫出計算 3 的倍數出現次數的迴圈，並分享表格。

### 3.5 20:35–20:45｜`while` 迴圈複習（10 分）
1. 透過 Notebook 第 3.5 節的表格，逐列說明「進入前的變數 → 判斷 → 更新後的變數」。
2. 強調條件必須會改變，否則容易無限迴圈；補充 `break` 與 `continue` 的作用。
3. 執行範例 code cell，觀察 `history` 清單中紀錄的 `before`、`condition`、`after` 值；請學生仿造寫出自己的 `while` 例子並用表格記錄。

### 4. 20:45–21:20｜密碼錯誤次數限制逐步建構（35 分）
拆解為四個階段，每個階段完成後檢查：
1. **階段 A — 計數器骨架**  
   - 給予範例：`max_tries = 5`, `tries = 0`。  
   - 學生練習：寫出 `while tries < max_tries` 的空迴圈，內部單純 `tries += 1`。
2. **階段 B — 結合密碼判斷**  
   - 將上週的 `user_input != password` 加入條件：`while user_input != password and tries < max_tries`。  
   - 請學生填表：每輪 `tries` 值、`user_input` 是什麼。
3. **階段 C — 剩餘次數提示**  
   - 引導學生計算 `remaining = max_tries - tries`，輸出訊息。  
   - 討論剩 1 次與剩 0 次時應顯示的文案。
4. **階段 D — 結束訊息與鎖定邏輯**  
   - 補上迴圈結束後的 `if` 判斷。  
   - 強調：若 `tries == max_tries` 代表失敗，要鎖定；否則就歡迎登入。
5. 補充 Notebook「4.0 邏輯拆解練習」真值表，檢查 `A = user_input != password`、`B = tries < max_tries - 1` 四種組合的判斷。
6. 若進度允許，加入延伸挑戰：使用 `for` 迴圈改寫、或加上 `break` 提前跳出。

### 5. 21:20–21:30｜回顧與課後任務
1. 請學生口頭說明今天最有收穫的概念與最大困惑。
2. 布置課後任務：完成 Notebook 中未填的表格，並撰寫一段流程圖或假碼描述密碼程式。
3. 約定下週檢核方式（例如現場小測或實作題）。

## 課後作業建議
- 完成 Notebook 內的所有填空與程式區塊，測試不同 `max_tries` 與密碼情境。
- 寫出至少 3 個日常情境，將它們轉成布林變數與真值表。
- 自主練習：改寫密碼程式，加入「倒數秒數」或「提示字數限制」等自訂條件。

## 備用方案與提醒
- 若布林語意轉換仍困難，可改採「卡片配對」活動：把語句與程式片段拆成卡片，讓學生配對。
- 如遇時間緊湊，可把階段 D 留到下週由學生完成；但務必在課堂中至少完成階段 B，建立信心。
