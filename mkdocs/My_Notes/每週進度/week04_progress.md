## 課程流程
### 19:30 — 開場與需求確認
1. 問學生是否有在家思考「錯誤五次鎖定」的寫法，學生坦言沒有進展，也不知道要問什麼。
2. 提出選項：直接寫密碼功能或先複習迴圈。學生說「想先複習」，因此調整流程。

### 19:35 — 迴圈暖身（耗時過長）
1. 一起寫下程式：
   ```python
   total = 0
   for i in range(10, 0, -1):
       total += i
       print(i)

   print("total =", total)
   ```
2. 問學生 `+=` 的意思與 `total += i` 的效果。學生口頭回答「懂」，但請學生轉換寫法`total = total + i`，卻寫不出來。下筆時固定寫成 `total += 1` 或 `total = total + 1`，顯示理解不足。
3. 鼓勵學生查資料、閱讀筆記以確認概念；同時示範如何使用數軸與表格紀錄每輪的 `i` 和 `total`。
4. 學生難以接受 `step = -1` 的倒數概念，把程式寫成 `range(10, 0, 1)`；反覆用例子、手勢與圖示協助。

### 20:15 — `range()` 詳解
1. 說明 `range(start, stop, step)` 的半開區間，為何 `stop` 不會出現在序列裡。
2. 以數軸比喻 `step` 正負號與大小；練習 `range(2, 12, 3)`、`range(5)`、`range(0, 10, 2)` 比較結果。
3. 加入 `len(range(...))`、索引與 `list(range(...))` 轉換，強調 `range` 本身是惰性的。

### 20:40 — 嘗試回到密碼題（未果）
1. 再度提到「最多五次錯誤」功能。學生仍無法動筆，或許「不知道從哪裡開始」。
2. 學生提問：要怎麼知道使用者輸入了幾次。
3. 改由教師提供提示：需要計數器、條件判斷與 `while`/`for` 迴圈；請學生先寫出判斷條件。學生無法組合 `(tries < MAX_TRIES)` 與 `user_input != password`。
4. 決定暫停主題，回頭補強布林邏輯。

### 21:00 — 布林與真值表補強
1. 回顧布林值 `True`/`False` 與 0/1 關係，說明 `and`、`or`、`not` 基本意義。
2. 建立二變數真值表與三變數括號示例（雖未深入），強調括號控制評估順序。
3. 播放範例後交付「自我填寫練習」：倒數追蹤表、條件拆解表、真值表小測驗，要求學生課後完成。

### 21:20 — 結尾
1. 再次確認學生理解狀況並說明下週計畫：從練習單檢討開始，再挑戰密碼題。
2. 課程延長到約 21:30，仍未完成任何延伸功能。
