# Week 7 後續延伸主題整理

> 這份筆記用於課後備課與學生自學安排，整理本週課程之後最適合銜接的六個知識點、導入重點、延伸練習與常見迷思。

## 1. 清單模擬佇列（Queue 模擬）
- **核心概念**：以 `list` 或 `collections.deque` 模擬 FIFO 佇列，理解 `head` / `tail` 指標與推入/彈出操作。
- **先備知識**：掌握 `list.append`, `pop(0)` 的成本差；認識 `deque` 的效率優勢。
- **引導流程**：
  1. 先讓學生手動在紙上模擬排隊情境，標註 `head` 與 `tail`。
  2. 實作簡易排隊程式，逐步印出佇列狀態。
  3. 分析固定大小佇列的「環狀」設計（對應檢測題 Q4）。
- **常見錯誤**：忽略 `tail` 會超出陣列末端、忘記重置計數器、搞混 `pop(0)` 與 `pop()`。
- **延伸活動**：改寫為 `deque`、加入「每三人送禮」或「跳號」規則觀察輸出變化。

## 2. 線性搜尋 vs. 二分搜尋比較
- **核心概念**：比較 `O(n)` 與 `O(log n)` 搜尋策略；追蹤 `low`, `mid`, `high` 隨每輪迴圈的變化。
- **先備知識**：熟悉排序後的列表、整數除法 `//`。
- **引導流程**：
  1. 用圖表或計數條示範線性搜尋與二分搜尋步驟。
  2. 撰寫兩個函式 (`f1`, `f2`)，記錄比較次數並印出。
  3. 練習分析最佳、最差、平均情境，呼應檢測題 Q6、Q22。
- **常見錯誤**：重複計算 `mid`、忘記更新 `low` / `high`、未處理未找到時的回傳值。
- **延伸活動**：用亂數生成資料測試執行次數、改成遞迴版本、加上統計圖表。

## 3. 費式與累加練習（迴圈版 DP 雛形）
- **核心概念**：以迴圈方式計算 Fibonacci 與前綴和陣列，強化「暫存上一輪結果」的想法。
- **先備知識**：熟悉變數更新順序、列表索引。
- **引導流程**：
  1. 先印出費式前幾項，讓學生手算並觀察規律。
  2. 實作迴圈版 Fibonacci，拆解 `temp` 交換用途（對應檢測題 Q7）。
  3. 建立前綴和陣列，計算區間加總，對應 Q30。
- **常見錯誤**：更新順序顛倒、漏掉初始值、在 for 迴圈裡改寫原陣列導致錯誤。
- **延伸活動**：比較遞迴與迴圈效能、加入模組化函式、嘗試動態輸入 N。

## 4. 遞迴觀念入門
- **核心概念**：辨識基本情況(Base Case)、遞迴呼叫與回傳路徑，避免無窮遞迴。
- **對應檢測題**：Q1（符號變化遞迴）、Q8（互呼遞迴）、Q11（階乘）、Q23（GCD）。
- **引導流程**：
  1. 練習畫遞迴呼叫樹，標出每個節點的回傳值。
  2. 從簡單的 `countdown` 與階乘入手。
  3. 解析互遞迴（`f1`/`f2`）的呼叫順序與輸出。
  4. 推導 Euclid 演算法：`GCD(a, b)` -> `GCD(b, a % b)`，觀察每次餘數變化。
- **常見錯誤**：缺少 base case、錯誤更新參數、忘記回傳值、把 print( ) 當回傳。
- **延伸活動**：實作階乘的尾遞迴版本、改寫成迴圈、觀察遞迴深度限制。

## 5. 條件覆蓋與圖形輸出
- **核心概念**：檢查 `if / elif / else` 的區間是否完整覆蓋；使用迴圈與格式化控制輸出圖案。
- **對應檢測題**：Q5（星號排列）、Q10（多層條件）、Q17（成績判斷）、Q19（字串格式化輸出）。
- **引導流程**：
  1. 讓學生列出每個條件所涵蓋的數值範圍。
  2. 改寫錯誤分數條件、補齊邊界 case。
  3. 拆解圖形題：先決定行數、空白數量，再組合字元。
  4. 練習用 `str.format` / `f-string` 生產固定寬度輸出。
- **常見錯誤**：條件順序錯誤、輸出多餘空白、`range` 終點抓錯。
- **延伸活動**：加上輸入驗證、模組化成函式、挑戰倒三角或菱形輸出。

## 6. 二維陣列掃描與鄰居計數
- **核心概念**：使用偏移列表遍歷四向或八向鄰居；處理邊界條件與統計資料。
- **對應檢測題**：Q18（迷宮鄰居計數）。
- **引導流程**：
  1. 從 3x3 的示意圖講解上/下/左/右偏移。
  2. 實作巢狀迴圈與 `dir` 列表，加入越界檢查。
  3. 觀察不同迷宮配置對統計結果的影響。
- **常見錯誤**：漏掉邊界檢查、對角線偏移寫錯、使用同一 `list` 當指標導致共享引用。
- **延伸活動**：改成 8 向鄰居、整合成 BFS/DFS 前置練習、配合 `collections.Counter` 做分類統計。

### BFS / DFS 基礎鋪墊
- **Breadth-First Search (BFS)**：使用佇列逐層擴散，適合找最短路徑或最少步數。流程是「取出節點 → 拓展鄰居 → 標記已訪問 → 依序加入佇列」。
- **Depth-First Search (DFS)**：使用堆疊（或遞迴呼叫堆疊）沿一條分支走到底，再回溯到上一層改走其他分支。適合統計區塊、判斷是否存在路徑等。
- **共通前置概念**：
  1. **訪問記錄**：使用 `visited` 陣列或在原地標記，避免重複拜訪。
  2. **鄰居取得**：可沿用前述 `dir` 偏移列表；在 BFS/DFS 中會重複使用。
  3. **資料結構選擇**：BFS 搭配 `collections.deque` 的 `popleft()`；DFS 可用遞迴或自建 `stack`。
  4. **停止條件**：設定目標節點或全部走訪完成後停止，避免無窮迴圈。
- **建議教學順序**：
  1. 從一維數列的「探索」練習過渡到二維網格。
  2. 先在紙上標示走訪順序，分別觀察 BFS 與 DFS 的差異。
  3. 實作「計算相連通道的面積」或「迷宮找出口」，逐行解釋佇列/遞迴的用法。
  4. 透過小迷宮示例比較兩種走訪順序並討論優缺點。
- **常見迷思澄清**：
  - BFS 並非只能用佇列，只要能先入先出的結構都可；但 `deque` 最方便。
  - DFS 遞迴等同於使用系統呼叫堆疊，若資料過大需改寫成迴圈搭配 stack。
  - 兩者都需要標記 visited；否則會在有環的圖上無限循環。
  - BFS 找到的第一個解通常是「最短步數」，DFS 則未必。
