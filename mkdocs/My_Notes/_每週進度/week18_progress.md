- 建議內容：
  - 版本 A：`math.gcd`（最快上手，重點放在輸入與題目拆解）。
  - 版本 B：輾轉相除法（強化邏輯與除法/取餘數觀念）。

### 2) 同一題用不同資料結構重寫（list / namedtuple / dataclass）
- 目的：讓學生「自己感受」可讀性、可維護性、出錯點差異，而不是只聽講解。
- 建議練習題材：
  - 「花店」小專題：例如花的 `name / price / stock`，支援進貨、販售、查詢、盤點、輸出報表。
  - 同一份需求，要求用：
    1. `list`（最直覺，但容易 index 搞混）
    2. `namedtuple`（不可變、欄位命名清楚）
    3. `dataclass`（可變、可加方法，適合狀態更新）

### 3) zip() 的 APCS 適用性評估
- 結論：**適用，但要「策略性使用」**。
- 常見適用情境（推薦教的）：
  - 並行走訪兩個 list：`for a, b in zip(A, B): ...`
  - 組合成配對資料：`pairs = list(zip(A, B))`
  - 轉置（2D 結構）：`zip(*matrix)`（列長一致時很乾淨）
- 不建議過度依賴的點（容易踩雷）：
  - `zip` 會「截斷到最短」：如果資料長度不一致，可能默默少算。
  - iterator 概念：`zip` 走過一次就消耗（初學者常混亂）。

### 3-1) map()（mapping）專項：輸入解析最常見的觀念斷點
- 症狀：學生看到 `map(int, input().split())` 會把它當成「已經是一個 list」，或不理解為何不能直接印、不能重複用。
- 需要釐清的 3 個核心點：
  1. `split()` 先把字串切成「很多個字串」。
  2. `map(int, ...)` 是把每個字串「映射」成整數（但回傳的是 iterator）。
  3. 要多次使用或要看到內容時，必須 `list(map(...))`，或直接 unpack：`a, b = map(int, input().split())`。
- 適合搭配的練習主題（建議一起教，效果最好）：
  - **輸入專項**：同一行多個數、固定筆數、多筆到 EOF（每一行 `map(int, line.split())`）。
  - **iterator 概念**：`map / zip` 都是 iterator（走一次就消耗），用「印兩次」的反例讓他記住。
  - **小型計算題**：`sum(map(int, input().split()))`、點積 `sum(a*b for a,b in zip(...))`（把 map 與 zip 串起來）。
  - **資料清理**：`map(str.strip, lines)`、`map(int, tokens)`（讓他知道 map 不只轉 int）。

### 4) 改錯題（Debug / Fix-it）納入講義的固定配方
- 方向：講義不只放「正確範例」，固定加入：
  - 讀輸入錯（unpack、空行、EOF、少讀/多讀）
  - 邏輯錯（少更新狀態、交換順序錯、邊界條件漏）
  - 資料結構錯（list index 用錯、欄位名打錯）
- 目的：訓練「觀察 → 定位 → 修正 → 解釋」能力。
