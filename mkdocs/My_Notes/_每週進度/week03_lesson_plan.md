# 第三週教案安排 W03 Lesson Plan

## 課前重點
- 確認學生已完成上週課後安排：LeetCode 任務、Python/VS Code 安裝、Jupyter Notebook 基本操作。
- 準備好教師端的 Python 環境與示範 Notebook（可沿用 `01_python_basics_warmup.ipynb`）。
- 預備 1 組 1–20 的數字小紙片與計時工具，用於實體排序複習與操作計數。
- 若學生仍未安裝 Python，事先準備離線安裝檔或官方下載頁面、VS Code Python 擴充套件連結。

## 教學目標
1. 學生能在個人筆電上啟動 Jupyter Notebook 並執行 Python 程式碼。
2. 透過 Notebook 實作氣泡排序，並加入步驟計數以觀察時間複雜度。
3. 分析氣泡排序的最佳、平均、最差情況，連結至 Big-O 表示法。
4. 介紹一個次要排序（插入排序或選擇排序），比較其操作流程與時間/空間複雜度。
5. 引導學生思考後續欲挑戰的 LeetCode 題目，建立課後練習方向。

## 所需材料與工具
- 學生與教師電腦（皆需可執行 Python 3 與 VS Code）。
- 教師準備好的 Jupyter Notebook 範例檔案（含排序骨架程式）。
- 數字小紙片 20 張（1–20），白板筆/色筆各 2 支。
- 網路連線（若需下載套件、登入 LeetCode）。
- 計時器（手機或線上工具皆可）。

## 課程流程（90 分鐘）

### 18:30–18:40｜開場與課後任務回顧
1. 關心學生是否完成上週任務，逐項追問：LeetCode 操作心得、Python 安裝結果、Notebook 使用狀況。
2. 若 Python 尚未安裝，使用教師預備的安裝流程協助完成；若已完成則請學生示範開啟 Notebook。
3. 重新確認課程目標：「今天要在你自己的電腦上寫出會計算步驟的排序程式」。

### 18:40–19:00｜Jupyter Notebook 建置與快速暖身
1. 帶學生在 VS Code 中建立全新 Notebook，命名為 `w03_sorting_lab.ipynb`。
2. 添加一個 Markdown Cell，整理今日學習目標與筆記框架。
3. 以簡單範例複習 `for` / `while` 迴圈結構，示範 `break` 如何提早結束回圈（例：找到第一個大於 10 的數字即停止）。
4. 新增 Code Cell，請學生輸入並執行：
   ```python
   import sys
   print('Hello, APCS!')
   print('Python version:', sys.version)
   ```
   - 若執行失敗，引導學生閱讀錯誤訊息、檢查環境。
5. 介紹 Notebook 的 `Shift+Enter` 執行、Kernel 重啟、儲存等基本操作。

### 19:00–19:20｜實體排序複習與計數
1. 使用小紙片快速回顧氣泡排序流程，請學生操作一次 8 張數字的排序並口頭敘述步驟。
2. 再次記錄比較次數、交換次數，與上週紀錄對照，確認學生理解 `n*(n-1)/2` 的由來。
3. 問：「如果資料量翻倍到 16，步驟數如何成長？」引導學生回答「約 4 倍」並連結 `O(n^2)`。

### 19:20–19:45｜Notebook 實作：氣泡排序 + 步驟計數
1. 於 Notebook 新增氣泡排序程式骨架，請學生逐行編寫：
   ```python
   def bubble_sort(arr):
       steps = 0
       swaps = 0
       n = len(arr)
       for i in range(n - 1):
           swapped = False
           for j in range(n - 1 - i):
               steps += 1  # 比較一次
               if arr[j] > arr[j + 1]:
                   arr[j], arr[j + 1] = arr[j + 1], arr[j]
                   swaps += 1
                   swapped = True
           if not swapped:  # 本輪無交換可提前結束
               break
       return arr, steps, swaps
   ```
2. 使用不同長度的測試資料（例如 8、12、16），讓學生記下 `steps` 與 `swaps`，並填入 Notebook 表格。
3. 示範帶有 `swapped` 旗標與 `break` 的最佳版本，請學生觀察最佳情況接近 `O(n)`。

### 19:45–20:05｜比較第二種排序並延伸分析
1. 介紹插入排序或選擇排序，簡述與氣泡排序的差異。
2. 請學生在 Notebook 中新增函式（可先提供骨架），並加入步驟計數邏輯。
3. 讓學生比較兩種排序在相同輸入下的 `steps`，討論：
   - 哪個演算法操作次數較少？
   - 兩者的 Big-O 是否不同？
   - 空間複雜度是否都屬於 `O(1)`？
4. 鼓勵學生描述最差與最佳情況各自的「故事」，強化直覺理解。

### 20:05–20:15｜整理與課後安排
1. Notebook 最後新增 Markdown Cell，請學生寫下今日收穫、疑問與下週想嘗試的 LeetCode 題目。
2. 一同規劃課後練習：
   - 在 LeetCode 搜尋排序相關 Easy 題目，嘗試用 Python 解題。
   - 若時間足夠，可在 Notebook 中紀錄嘗試過的題目與測試資料。
3. 說明下週預告：若環境穩定，將開始討論陣列以外的資料結構（例如 Stack 或 Tree）與對應題目。

## 課後作業建議
- 完成 Notebook 中尚未測試的輸入規模（例如 20、30），並觀察步驟數的成長趨勢。
- 於 Notebook 中加入簡單的圖表（使用 `matplotlib` 或手動表格），將 `n` 與 `steps` 的對應關係視覺化。
- 在 LeetCode 上選做 1 題排序或陣列操作的 Easy 題，記錄題號、解題思路與提交結果。
- 整理安裝 Python 過程可能遇到的問題與解法，供未來回顧或分享。

## 備用與彈性安排
- 若學生仍無法在個人電腦執行 Notebook，可改以教師電腦示範，並把剩餘時間撥給環境設定教學。
- 若氣泡排序理解度已高，可提前導入「計數排序」或「遞迴版本的排序」概念，對照時間/空間差異。
- 若進度超前，可預覽樹的基本概念：節點、兒子、遞迴遍歷，為下週鋪陳。

> 提醒：整堂課務必保留 5–10 分鐘整理教材與記錄學生狀態，維持連貫的教學紀錄。
