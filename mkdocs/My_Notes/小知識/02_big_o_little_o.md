# Big-O 與 little-o 入門指南

這篇文章緊接〈時間與空間複雜度入門〉，專注釐清 Big-O（`O`）與 little-o（`o`）的差異，提供直覺比喻與程式例子，幫助初學者建立漸進分析的語感。

## 先複習：Big-O 是什麼？
- `O(f(n))` 表示「成長率不會超過 `f(n)` 的某個常數倍」，強調上界（upper bound）。
- 例：`T(n) = 3n^2 + 5n + 8` ⇒ `T(n) ∈ O(n^2)`，因為當 `n` 夠大時，`3n^2` 會壓過其他項。
- Big-O 可以包含與 `f(n)` 同級的項目，甚至比 `f(n)` 小的項目。

## 那 little-o 呢？
- `o(f(n))` 意指：當 `n → ∞`，`T(n)` 成長得「嚴格慢於」 `f(n)`，也就是 `T(n) / f(n) → 0`。
- 小寫的 `o` 排除了與 `f(n)` 同級的項，只允許更低階的成長率。

### 比喻
- 想像 `f(n)` 是高鐵，`T(n)` 是普通自強號。  
  若 `T(n) ∈ O(f(n))`，代表普通車不會比高鐵慢太多（可能同速或稍慢）。  
  若 `T(n) ∈ o(f(n))`，代表普通車永遠追不上高鐵，兩者速度差距會越來越大。

## 例子對照

| 函式 `T(n)` | `O(n^2)`? | `o(n^2)`? | 說明 |
| --- | --- | --- | --- |
| `n^2` | ✅ | ❌ | 成長率同級，只能說是 `O(n^2)` |
| `5n log n` | ✅ | ✅ | 比 `n^2` 慢很多，兩者比值 → 0 |
| `100n^2 + 1` | ✅ | ❌ | 仍有 `n^2` 項，無法屬於 `o(n^2)` |
| `n^{1.5}` | ✅ | ✅ | 次方較低，屬於 `o(n^2)` |
| `n^3` | ❌ | ❌ | 超過 `n^2` 上界 |

### Python 觀察

```python
import math

def ratio(values, f):
    points = [10, 100, 1000, 10_000]
    return [values(n) / f(n) for n in points]

print('n^2 對 n^2:', ratio(lambda n: n**2, lambda n: n**2))
print('n log n 對 n^2:', ratio(lambda n: n * math.log2(n), lambda n: n**2))
print('n^(1.5) 對 n^2:', ratio(lambda n: n**1.5, lambda n: n**2))
```

- 第一列比值趨近 1（常數），說明 `n^2` 只是 `O(n^2)`。
- 後兩列比值趨近 0，符合 `o(n^2)` 的條件。

## 實際應用

### 1. 判斷演算法改進是否成功
假設原始解法 `T_old(n) = 2n^2 + 10n`，新解法 `T_new(n) = 50n log n`。

- `T_new ∈ O(n^2)` 仍成立，但這樣的描述無法凸顯「新解法更快」。
- 若能說 `T_new ∈ o(n^2)`，就表示新解法的成長率嚴格低於平方，不會退化回平方級。

### 2. APCS／LeetCode 場景
- 當題目提示「盡量做到 `O(n log n)`」時，若你的演算法屬於 `o(n^2)`，代表已經明顯優於平方時間，通常能通過測資。
- 在寫解析時，可以精確說明：`T(n) = n log n + 3n` ⇒ `T(n) ∈ O(n log n)`，同時 `T(n) ∈ o(n^2)`，讓讀者更容易理解效能優勢。

## 常見迷思澄清
1. **Big-O 不是唯一描述**：同一個函式可以屬於多個集合。例如 `n` 同時在 `O(n)`, `O(n log n)`, `O(n^2)`……，只是常見會選擇最緊的上界來表示。
2. **little-o 不是「很小」**：它是一個數學符號，代表成長率嚴格小於，和中文字面意義無關。
3. **不要忽略常數的存在**：雖然 Big-O 掩蓋常數，但在實際測試中常數仍有影響。分析時可以先寫出精確式子，再視情況轉換成 Big-O 或 little-o。

## 延伸：還有哪些符號？
- `Ω(f(n))`：下界（至少需要這麼多）。
- `Θ(f(n))`：上下界一致，成長率緊密貼合。
- `ω(f(n))`：下界的嚴格版本（成長率嚴格大於）。

這些符號共同構成漸進分析的語言，讓我們能精確描述演算法行為。

## 小挑戰
判斷下列敘述是否正確：

1. `n log n ∈ o(n)`
2. `n ∈ o(n log n)`
3. `n^{1.1} ∈ O(n)`
4. `n^2 ∈ o(n^2 log n)`

**參考解答**

1. ❌（`n log n` 比 `n` 成長更快）
2. ✅（`n / (n log n) = 1 / log n → 0`）
3. ❌（`n^{1.1}` 次方較大，非 `O(n)`）
4. ✅（`n^2 / (n^2 log n) = 1 / log n → 0`）

## 小結
- Big-O 給你「不會超過這樣」的保證，而 little-o 表示「成長率會明顯更低」。
- 使用 little-o 能更精確地說明演算法改進的幅度。
- 先熟悉 Big-O，再用範例（如 `n^2` vs `n log n`）練習分辨何時可以進一步使用 little-o。熟悉這些符號後，面對 APCS 或線上題庫時，就能更自信地分析效能並撰寫清楚的解題報告。
