# APCS 題目拆解流程

面對 APCS 或 LeetCode 題目時，與其直接開寫程式，不如按步驟拆解。這份流程筆記提供一套可重複使用的思考框架。

## Step 1. 讀題與重述
- 仔細閱讀題目、範例輸入輸出。
- 自己用白話重述一次：「題目要我做什麼？輸入輸出格式是？」
- 找到關鍵限制（`n` 最大多少、資料型態、是否有負數）。

### 小技巧
- 把輸入輸出寫在紙上或註解中，一邊確認格式。
- 若題目提供多組測試資料，先抓至少一組做手算。

## Step 2. 找出對應類型
- 類型範例：排序、字串處理、模擬、資料結構、圖論。
- 問自己：這題像之前哪題？有無標準做法？
- 確認時間複雜度需求（例如：`n` 最多 `10^5`，就不能寫 `O(n^2)`）。

## Step 3. 規劃資料結構與演算法
- 選擇資料結構（`list`／`dict`／`set`／`queue`...）。
- 寫下演算法步驟：
  1. 讀取輸入。
  2. 處理資料（排序／累加／搜尋）。
  3. 產生輸出。
- 可先寫成偽程式碼：

```text
for 每個學生：
    計算總分
依總分排序（降冪）
輸出姓名與總分
```

## Step 4. 寫程式草稿
- 先把 I/O 框架寫好，再填入邏輯。
- 避免一次寫完所有功能，建議分段測試（例如先確認輸入讀取正確）。
- 加入必要的 `assert` 或暫時打印輔助除錯。

## Step 5. 自我測試
- **最小案例**：`n = 1`、空字串、全相同值。
- **一般案例**：題目提供的範例，或自己設計常見輸入。
- **極端案例**：最大輸入量、邊界條件（負數、0、重複）。
- 若時間允許，可寫自動化測試或比較兩種解法的輸出是否一致。

### 範例：成績排名題

```python
def summarize(records):
    results = []
    for name, scores in records:
        total = sum(scores)
        results.append((name, total))
    results.sort(key=lambda x: x[1], reverse=True)
    return results

sample = [
    ("Aaron", [80, 90, 85]),
    ("Kevin", [90, 92, 88]),
]

print(summarize(sample))
```

- 小測試：新增一位總分相同的學生，確認排序是否穩定。

## Step 6. 檢查輸出格式
- 是否需要換行、空格、小數位數？
- 是否要輸出多組資料之間的空行？
- 常見錯誤：多印空格、少換行、大小寫不符。

## Step 7. 優化與整理
- 提交前掃描程式：命名是否清楚？是否留了多餘 `print`？
- 回到複雜度觀念：`O(n log n)` 是否符合要求？是否可再降為 `O(n)`？

## 附錄：常見錯誤清單
- 忘記把字串轉成整數。
- 沒有處理空輸入或 `n = 0`。
- 巢狀迴圈索引範圍寫錯（例如 `range(len(nums) - 1)`）。
- 排序方向弄反（缺少 `reverse=True`）。

## 小結
- 每次照流程走，可以讓你逐步建立「題目 → 步驟 → 程式 → 驗證」的肌肉記憶。
- 久而久之，你會更快定位問題出在哪一步，也能把解題心得寫成清楚的筆記。
- 建議把這份流程列印或複製到自己的學習檔案中，上課或自習時隨時翻看。
