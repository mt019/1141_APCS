## 1. 輸入與變數命名

### 1.1 為何要把 `input()` 存進變數？
- `input()` 回傳字串，若不存起來就無法在後續判斷使用。
- 變數命名要清楚區分「實際密碼」與「使用者輸入」，避免覆寫。

```python
password = "1234"                 # 設定預期密碼
user_input = input("請輸入密碼：")  # 將輸入保存到變數
```

#### 常見錯誤
```python
password = input("請輸入密碼：")  # ❌ 這會把原本設定好的 password 覆蓋掉
```

### 1.2 什麼時候可以覆蓋變數？
- 對同一個變數「重新指定」表示保留變數名稱、更新內容。
- 在重試流程中，可以用相同的變數名稱保存最新輸入。

```python
password = "1234"
user_input = input("請輸入密碼：")

while user_input != password:
    user_input = input("密碼錯誤，再試一次：")  # ✅ 更新為最新輸入
```

## 2. `while` 迴圈：加入錯誤次數限制

### 2.1 基本流程
1. 先取得第一次輸入。
2. 當前輸入不正確時進入迴圈。
3. 迴圈內提示錯誤、更新輸入，並累計錯誤次數。
4. 超過上限時結束，避免無限次嘗試。

```python
MAX_TRIES = 5
password = "1234"
tries = 0

user_input = input("請輸入密碼：")

while user_input != password and tries < MAX_TRIES - 1:
    tries += 1
    print(f"密碼錯誤！已錯 {tries} 次")
    user_input = input("請再試一次：")

if user_input == password:
    print("密碼正確，歡迎光臨")
else:
    print("輸入錯誤次數過多，請稍後再試")
```

> 小提示：先判斷「是否還有重試機會」，避免在迴圈裡直接 `break`，程式更好讀。

- **練習 C**：將提示訊息抽成函式，例如 `show_hint(count, limit)`。

## 3. 迴圈控制關鍵字：`break`、`continue`、`return`

### 3.1 `break`：提前結束迴圈
- 終止「目前」的 `for` 或 `while` 迴圈，不會再執行迴圈內或下一次的內容。
- 常用於找到答案後立即跳出。

```python
attempts = 0
while attempts < 5:
    user_input = input("請輸入密碼：")
    attempts += 1

    if user_input == "1234":
        print("成功登入")
        break  # 找到正確答案就結束迴圈

    print("密碼錯誤")
else:
    print("輸入錯誤次數過多")
```

> 注意：`while ... else` 只有在迴圈「正常結束」時才會執行。若使用 `break` 跳出，`else` 區塊會被略過。

### 3.2 `continue`：跳過當次迴圈剩餘程式
- 中斷本次迭代，直接進入下一輪。
- 適合在特定條件下「略過」後續動作，但保留整個迴圈流程。

```python
for pwd in ["0000", "1111", "1234", "4321"]:
    if pwd == "1234":
        continue  # 略過已知正確密碼
    print("嘗試預設密碼：", pwd)
```

### 3.3 `return`：離開函式（可能在迴圈內發生）
- `return` 不僅離開迴圈，還會結束整個函式，並可帶回回傳值。
- 寫在函式外（頂層）時會產生語法錯誤；請確認 `return` 位置。

```python
def verify(password):
    while True:
        user_input = input("請輸入密碼：")
        if user_input == password:
            return True   # 成功就離開函式
        if user_input == "quit":
            return False  # 使用者放棄
        print("再試一次，或輸入 quit 離開")
```

### 3.4 練習題
- **練習 D**：在密碼程式中加入「輸入 `quit` 時提前結束」的功能，試著使用 `break` 與 `return` 實作兩個版本。
- **練習 E**：請學生說明 `break` 與 `return` 在函式中的差異，並舉出各自適用的情境。
- **練習 F**：改寫迴圈，使用 `continue` 略過特定錯誤（例如空字串）再提示使用者。

## 4. 函式呼叫與值傳遞

### 4.1 概念說明
- Python 會將「運算後的結果」傳入函式，但不會自動改變外面的變數。
- 在 APCS 題目中，`f2(m + 2)` 不會影響 `m` 本身的值。

```python
def add_two(x):
    print("函式內的 x =", x)
    return x + 2

m = 3
new_value = add_two(m + 2)  # 傳入 5
print("函式呼叫完 m =", m)  # 仍然是 3
```

- 只有在函式內明確回傳並重新指派，才會改變原來的變數值。

```python
m = add_two(m)  # 現在 m 會變為 5
```

## 5. 遞迴追蹤工具

### 5.1 追蹤表格示範
| 呼叫層級 | 函式 | 參數值 | 動作 | 備註 |
|---------|------|--------|------|------|
| 1       | f1   | m=1    | 進入 | m <= 3，需呼叫 f2 |
| 2       | f2   | n=3    | 進入 | n <= 3，呼叫 f1(2) |
| 3       | f1   | m=2    | 進入 | ... | ... |

> 建議學生在紙上畫出遞迴樹或呼叫堆疊，理解哪個呼叫先完成、哪個後完成。

### 5.2 `countdown` 範例再練

```python
def countdown(n):
    if n <= 0:
        print("Lift off!")
        return
    print(n)
    countdown(n - 1)

countdown(3)
```

- 請學生回答：
  1. `countdown(3)` 共有幾層呼叫？
  2. 何時會印出 `Lift off!`？
  3. 若 `n = -1`，程式會發生什麼事？
