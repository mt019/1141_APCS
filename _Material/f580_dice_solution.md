# f580. 2. 骰子 — 解題講解筆記

本文件用於課堂解題說明：先鋪墊必要知識點與資料結構，再進入解題流程與實作重點。

---

## 一、必要知識點（先備）

1. **模擬（simulation）**
   - 按照題目規則逐步更新狀態，不需要找數學公式。
   - 尤其是「旋轉、交換」這種明確操作，最適合模擬。

2. **狀態表示**
   - 一個物件的狀態可以用多個變數/結構描述。
   - 本題每顆骰子的狀態 = 六個面的點數如何排列。

3. **陣列/列表的應用**
   - 用 list 表示骰子各面，例如：
     - `top, bottom, front, back, right, left`
   - 旋轉就是這六個值的重新對應。

4. **索引與交換**
   - 若 `a, b` 都是正整數 → 交換第 `a` 顆與第 `b` 顆骰子。
   - 要能在 O(1) 時間內完成交換。

---

## 二、資料結構設計

### 1) 單顆骰子的表示
用固定順序保存六個面：

```
index: 0    1      2      3     4     5
face : top bottom front back right left
```

例：
- 初始狀態為：top=1, front=4, right=2
- 那麼 bottom=6, back=3, left=5（相對面點數和為 7）

可以用 list：

```
dice = [1, 6, 4, 3, 2, 5]
```

### 2) 多顆骰子的表示
用 list 存所有骰子：

```
dices = [dice1, dice2, ..., diceN]
```

交換操作只需要 swap list 的兩個元素即可。

---

## 三、核心操作：旋轉規則

### 1) 向前旋轉（b = -1）
- top → front
- front → bottom
- bottom → back
- back → top
- left/right 不變

### 2) 向右旋轉（b = -2）
- top → right
- right → bottom
- bottom → left
- left → top
- front/back 不變

這兩個旋轉是本題關鍵，建議直接寫成函式。

---

## 四、解題流程（高層邏輯）

1. 讀入 `n, m`
2. 初始化 `n` 顆骰子（全部同一個初始狀態）
3. 重複 `m` 次操作：
   - 若 `a, b > 0` → 交換第 `a` 顆與第 `b` 顆
   - 若 `b == -1` → 對第 `a` 顆做「向前旋轉」
   - 若 `b == -2` → 對第 `a` 顆做「向右旋轉」
4. 輸出所有骰子最上面的點數

---

## 五、易錯點提醒

1. **索引是 1-based**
   - 題目編號從 1 開始，Python list 從 0 開始。

2. **旋轉規則方向不可弄反**
   - 要先畫出骰子展開圖確認位置。

3. **交換後骰子狀態要整組交換**
   - 交換的是整顆骰子，不是只交換 top。

---

## 六、時間與空間複雜度

- 每次操作都是 O(1)
- 總時間 O(m)
- 空間 O(n)

## 八、簡易伪碼（教學版）

```
init dice = [1,6,4,3,2,5]
all_dice = [copy(dice) for i in range(n)]

for each (a, b):
    if b > 0:
        swap(all_dice[a-1], all_dice[b-1])
    else if b == -1:
        roll_forward(all_dice[a-1])
    else if b == -2:
        roll_right(all_dice[a-1])

print all_dice[i][0]
```
